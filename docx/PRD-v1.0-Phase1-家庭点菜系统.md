## 1. 产品概述

### 1.1 阶段目标

第一阶段为 MVP（最小可行产品），目标：

1. 实现客人端基本点菜、下单、虚拟支付功能
2. 实现厨师端菜品管理、订单状态流转功能
3. 完成基础数据模型与 API

### 1.2 产品愿景

**家庭点菜系统**是一个面向家庭或小型朋友圈的餐饮服务工具。通过微信小程序，客人可以浏览当日菜单、点菜并完成支付；厨师通过 Web 后台管理菜品和订单。

## 3. 功能需求

### 3.1 客人端（微信小程序）

#### 3.1.1 用户授权与登录

**功能描述**：首次打开小程序时，获取微信用户昵称、头像，并绑定 openId 作为唯一标识。无需额外注册，自动创建客人账户。

**功能详单**：

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 微信授权 | 获取昵称、头像、openId | P0 |
| 自动注册 | openId 不存在时自动创建账户 | P0 |

#### 3.1.2 菜单浏览

**功能描述**：展示当日可点菜品列表，支持按分类筛选。每个菜品显示：图片、名称、价格、简介、是否售罄。

**功能详单**：

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 分类筛选 | 按凉菜、热菜、主食、饮料等分类筛选 | P0 |
| 菜品列表 | 展示今日供应的菜品 | P0 |
| 售罄标识 | 售罄菜品显示不可点击状态 | P0 |

#### 3.1.3 点菜与购物车

**功能描述**：点击菜品可查看详情，并选择份数加入购物车。购物车内可修改份数、删除菜品，实时计算总价。

**功能详单**：

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 加入购物车 | 选择份数后加入 | P0 |
| 购物车列表 | 显示已选菜品和份数 | P0 |
| 修改份数 | 增加或减少菜品数量 | P0 |
| 删除菜品 | 从购物车移除菜品 | P0 |
| 价格汇总 | 实时计算总价 | P0 |

#### 3.1.4 下单

**功能描述**：从购物车进入确认订单页，可填写备注，提交订单后生成待支付订单。

**功能详单**：

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 订单确认页 | 显示菜品列表、总价、备注输入框 | P0 |
| 备注功能 | 支持输入特殊要求（如少辣、不要香菜） | P1 |
| 提交订单 | 生成待支付订单 | P0 |
| 订单号 | 生成唯一订单号 | P0 |

#### 3.1.5 支付

**功能描述**：采用虚拟支付模式，点击"确认支付"即视为支付成功。

**功能详单**：

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 虚拟支付 | 点击确认支付，状态变更为已支付 | P0 |

#### 3.1.6 订单管理

**功能描述**：我的订单列表展示所有历史订单，按时间倒序。订单详情显示当前状态，支持取消待支付订单。

**功能详单**：

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 订单列表 | 展示所有历史订单，按时间倒序 | P0 |
| 订单状态展示 | 显示待支付、已支付、制作中、已完成、已取消 | P0 |
| 订单详情 | 查看订单完整信息（菜品、金额、备注） | P0 |
| 取消订单 | 取消待支付订单 | P0 |

### 3.2 厨师端（Web后台）

#### 3.2.1 登录

**功能描述**：厨师使用账号密码登录 Web 后台。

**功能详单**：

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 账号密码登录 | 输入用户名密码登录 | P0 |
| JWT 会话 | 登录成功后获取 JWT Token | P0 |

#### 3.2.2 分类管理

**功能描述**：管理菜品分类，如凉菜、热菜、汤类、主食、饮料等。

**功能详单**：

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 分类列表 | 查看所有分类 | P0 |
| 创建分类 | 添加新分类 | P0 |
| 编辑分类 | 修改分类名称 | P0 |

#### 3.2.3 菜品管理

**功能描述**：管理菜品信息，包括增删改查、设置售罄、设置今日供应。

**功能详单**：

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 菜品列表 | 展示所有菜品，支持分类筛选、名称搜索 | P0 |
| 添加菜品 | 填写名称、分类、价格、图片、简介 | P0 |
| 编辑菜品 | 修改菜品信息 | P0 |
| 设置售罄 | 单个菜品标记为售罄 | P0 |
| 设置今日供应 | 单个菜品标记为今日供应 | P0 |

#### 3.2.4 订单管理

**功能描述**：查看和处理订单，跟踪订单状态流转。

**功能详单**：

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 订单列表 | 默认展示当天订单，支持状态筛选 | P0 |
| 订单详情 | 查看订单完整信息（菜品明细、备注） | P0 |
| 状态流转 | 更新订单状态（开始制作、完成） | P0 |

**订单状态流转**：

```
┌─────────────┐    客人支付    ┌─────────────┐
│ pending_    │ ────────────→ │    paid     │
│ payment     │                │ (待制作)    │
│ (待支付)    │                │             │
└─────────────┘                └─────────────┘
       │                              │
       │ 客人取消                      │ 厨师点击"开始制作"
       ▼                              ▼
┌─────────────┐                ┌─────────────┐
│ cancelled  │                │ preparing  │
│ (已取消)   │                │ (制作中)   │
└─────────────┘                └─────────────┘
                                        │
                                        │ 厨师点击"完成"
                                        ▼
                                 ┌─────────────┐
                                 │ completed  │
                                 │ (已完成)   │
                                 └─────────────┘
```

## 4. 数据模型

### 4.1 实体关系图

```
┌──────────┐       ┌──────────┐       ┌──────────┐
│   User   │──────→│  Order   │←──────│  Dish    │
│  （用户） │  1:N  │ （订单）  │  N:1  │ （菜品）  │
└──────────┘       └──────────┘       └──────────┘
                        │                    │
                        │ 1:N                │ 1:N
                        ▼                    ▼
                 ┌──────────┐         ┌──────────┐
                 │OrderItem │         │ Category │
                 │（订单项） │         │ （分类）  │
                 └──────────┘         └──────────┘
```

### 4.2 核心表结构

#### users（用户表）

| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| id | int | 主键 | PK, AUTO_INCREMENT |
| openId | varchar(64) | 微信唯一标识 | UNIQUE, NOT NULL |
| nickname | varchar(50) | 微信昵称 | |
| avatarUrl | varchar(255) | 微信头像 | |
| role | enum | 角色：admin/merchant/user | DEFAULT 'user' |
| disabled | boolean | 是否禁用 | DEFAULT false |
| createdAt | datetime | 创建时间 | |
| updatedAt | datetime | 更新时间 | |

#### categories（分类表）

| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| id | int | 主键 | PK, AUTO_INCREMENT |
| name | varchar(50) | 分类名称 | NOT NULL |
| sortOrder | int | 排序序号 | DEFAULT 0 |
| createdAt | datetime | 创建时间 | |

#### dishes（菜品表）

| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| id | int | 主键 | PK, AUTO_INCREMENT |
| categoryId | int | 分类ID | FK → categories.id |
| name | varchar(100) | 菜品名称 | NOT NULL |
| price | decimal(10,2) | 价格 | NOT NULL |
| image | varchar(255) | 图片URL | |
| description | text | 简介 | |
| available | boolean | 是否在售（全局） | DEFAULT true |
| todaySupply | boolean | 今日是否供应 | DEFAULT false |
| createdAt | datetime | 创建时间 | |

#### orders（订单表）

| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| id | int | 主键 | PK, AUTO_INCREMENT |
| orderNo | varchar(32) | 订单号 | UNIQUE, NOT NULL |
| userId | int | 用户ID | FK → users.id |
| totalAmount | decimal(10,2) | 总金额 | NOT NULL |
| status | enum | 订单状态 | 见状态流转 |
| remark | text | 备注 | |
| createdAt | datetime | 创建时间 | |
| updatedAt | datetime | 更新时间 | |

**订单状态（status）**：

| 状态值 | 说明 |
|--------|------|
| pending_payment | 待支付 |
| paid | 已支付（待制作） |
| preparing | 制作中 |
| completed | 已完成 |
| cancelled | 已取消 |

#### order_items（订单项表）

| 字段名 | 类型 | 说明 | 约束 |
|--------|------|------|------|
| id | int | 主键 | PK, AUTO_INCREMENT |
| orderId | int | 订单ID | FK → orders.id |
| dishId | int | 菜品ID | FK → dishes.id |
| quantity | int | 数量 | NOT NULL |
| price | decimal(10,2) | 下单时的单价（快照） | NOT NULL |

### 6.2 项目结构

```
src/
├── modules/
│   ├── auth/           # 认证模块：JWT 登录、微信登录
│   ├── user/           # 用户管理：CRUD、角色管理
│   ├── order/          # 订单管理：创建、状态流转
│   ├── dish/           # 菜品管理：CRUD
│   ├── category/       # 分类管理
│   └── events/         # WebSocket：实时通知
├── common/
│   ├── guards/         # JwtAuthGuard、RolesGuard
│   ├── filters/        # HttpExceptionFilter
│   ├── interceptors/   # TransformInterceptor、LoggingInterceptor
│   └── decorators/     # @Roles()、@CurrentUser()、@Public()
├── config/             # 配置文件
├── database/
│   └── migrations/     # 数据库迁移
```
