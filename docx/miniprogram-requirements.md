# 私人厨师点餐小程序 - 需求文档

## 1. 项目概述
- **项目名称**：私厨点餐助手
- **技术栈**：Taro 4.0 + React + TypeScript + Taro UI
- **目标用户**：私厨（厨师/管理员）与客人（家庭成员或朋友）
- **核心价值**：私厨可发布当日菜品、管理订单；客人可浏览菜单、点餐、查看订单状态，降低家庭点餐沟通成本。

## 2. 角色与页面权限
| 角色   | 可访问页面                           | 说明                         |
|--------|--------------------------------------|------------------------------|
| 私厨   | 菜品管理、订单管理、客人管理、统计看板 | 通过登录态识别，首次需登录   |
| 客人   | 菜单浏览、购物车、下单、我的订单       | 通过微信授权自动登录         |

> 注：小程序启动时根据本地存储的 role 或后端返回的权限决定跳转 TabBar 或重定向。

## 3. 客人端功能详解

### 3.1 授权与登录
- 调用 `wx.login` 获取 code，后端换取 openId 并生成自定义登录态。
- 首次登录需获取微信昵称、头像，自动创建用户。
- 后续打开小程序自动静默登录，保持会话。

### 3.2 菜单浏览页（首页）
- **展示内容**：仅展示今日供应（`todaySupply = true`）且未售罄的菜品。
- **筛选**：顶部 Tab 按分类（凉菜、热菜、主食、饮料等）切换，支持横向滑动。
- **菜品卡片**：菜品图片、名称、价格、简介、售罄标记。
- **操作**：点击卡片进入详情页；底部悬浮“购物车”入口，显示当前购物车总数量。

### 3.3 菜品详情页
- 大图展示、名称、价格、详细介绍。
- 数量选择器（步长1，最大可设上限）。
- 加入购物车按钮（反馈 toast 提示）。

### 3.4 购物车页
- 列出所有已加菜品，支持修改数量、删除、清空。
- 实时计算总金额。
- “去结算”按钮跳转至确认订单页。
- 购物车数据暂存于本地（storage）或服务端（建议本地，减少请求）。

### 3.5 确认订单页
- 展示订单明细（菜品、数量、单价、小计）、总金额。
- 备注输入框（选填，如“少辣”）。
- 提交订单 → 生成订单（状态为“待支付”），跳转至订单详情页并提示支付。

### 3.6 支付页（虚拟支付）
- 订单详情页展示订单信息，点击“立即支付”按钮。
- 虚拟支付：直接调用后端支付成功接口，订单状态变更为“已支付/待制作”。
- 支付成功后跳转至“我的订单”列表。

### 3.7 我的订单页
- 按时间倒序展示所有订单，支持按状态筛选（全部、待支付、待制作、制作中、已完成、已取消）。
- 订单卡片显示：订单号、总金额、状态、下单时间。
- 点击进入订单详情页，可查看菜品明细、备注。
- 待支付订单可点击“取消订单”。

### 3.8 个人中心页
- 展示微信头像、昵称。
- 显示累计消费金额、订单总数。
- 提供“联系私厨”入口（可配置客服或电话）。

## 4. 私厨端功能详解

### 4.1 登录页
- 账号密码登录（后端预设初始账号，支持修改密码）。
- 登录成功后存储 token，跳转至管理首页。

### 4.2 菜品管理页
- **分类管理**：可增删改分类（名称、排序），分类列表展示。
- **菜品列表**：以卡片形式展示，支持按分类筛选、搜索。
  - 每个菜品可编辑、删除、切换“今日供应”开关。
- **新增/编辑菜品**：表单包含名称、分类、价格、图片上传（使用 Taro.chooseImage）、简介、售罄开关。
- **批量设置今日供应**：多选菜品，一键设为今日供应。

### 4.3 订单管理页
- **订单列表**：默认展示今日订单，可筛选日期、状态。
  - 列表项：订单号、客人昵称、总金额、状态、操作按钮。
- **订单详情弹窗/页**：展示完整订单信息（菜品明细、备注）。
  - 状态流转按钮：
    - 已支付/待制作 → “开始制作”（状态变更为制作中）
    - 制作中 → “完成制作”（状态变更为已完成）
    - 已完成不可操作；待支付订单客人可取消，私厨不可操作。

### 4.4 客人管理页
- 显示所有客人列表（昵称、头像、注册时间、累计消费金额）。
- 可禁用某客人（禁用后该客人无法下单）。
- 支持搜索客人。

### 4.5 统计看板（可选 MVP）
- 今日订单总数、今日营收。
- 近7日订单趋势图（简单柱状图）。
- 热门菜品 Top5（按销量）。

## 5. 页面路由与 TabBar 设计

### 5.1 客人端 TabBar
- **菜单**（`/pages/menu/index`）
- **购物车**（`/pages/cart/index`）
- **我的订单**（`/pages/orders/index`）
- **我的**（`/pages/profile/index`）

### 5.2 私厨端（无 TabBar，通过侧边栏或底部导航切换）
- **菜品管理**（`/pages/chef/dish/index`）
- **订单管理**（`/pages/chef/order/index`）
- **客人管理**（`/pages/chef/user/index`）
- **统计看板**（`/pages/chef/stat/index`）

> 私厨端可单独构建一套页面，也可通过权限控制显示不同菜单。建议使用不同入口或动态配置 TabBar。

## 6. 交互与体验要求
- 所有请求需显示 loading 状态，错误需友好提示。
- 下单、支付、状态变更等关键操作需二次确认（Modal）。
- 购物车实时刷新总价，支持空状态提示。
- 图片加载使用懒加载，默认占位图。
- 适配不同屏幕尺寸，使用 rpx 单位。

## 7. 数据交互约定
- 所有 API 遵循 RESTful 风格，请求前缀为 `/api`。
- 统一响应格式：
  ```json
  {
    "code": 0,
    "message": "success",
    "data": {}
  }
 - 登录态通过 Authorization: Bearer <token> 传递（私厨端）；客人端使用 openId 作为身份凭证（由后端维护会话）。
 - 小程序端需实现请求拦截器处理 token 过期、自动刷新等。
##  8. 非功能性需求
- 首屏加载时间 < 2秒（使用分包加载）。

- 支持 iOS 与 Android 微信版本兼容性。-

- 代码通过 ESLint + Prettier 统一格式。

- 预留后端接口切换环境配置（开发、测试、生产）。

## 9. 迭代规划
- V1.0：实现客人端完整点餐流程 + 私厨端基础管理（菜品、订单、客人）。

- V1.1：增加统计看板、批量设置今日供应。

- V2.0：接入微信支付、订阅消息通知。
