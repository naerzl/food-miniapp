## 1. 产品概述

### 1.1 产品愿景

**家庭点菜系统**是一个面向家庭或小型朋友圈的餐饮服务工具。通过微信小程序，客人可以浏览当日菜单、点菜并完成支付；厨师通过 Web 后台管理菜品和订单。系统以极简方式满足家庭内部点餐需求，降低沟通成本，增加用餐趣味性。

### 1.2 产品定位

- **目标用户**：家庭成员、小型朋友圈（3-10人）
- **使用场景**：日常家庭用餐、朋友聚餐
- **核心痛点**：口头点菜容易遗漏、不知道家人想吃什么、无法跟踪订单状态

### 1.3 核心价值主张

| 价值 | 说明 |
|------|------|
| 自助点餐 | 客人自主浏览菜单下单，减少口头沟通 |
| 状态透明 | 实时跟踪订单状态（待支付→制作中→已完成） |
| 数据驱动 | 为厨师提供菜品排行、消费统计等数据洞察 |
| 趣味体验 | 增加用餐趣味性，支持家庭内部"记账"模式 |

## 2. 用户角色

### 2.1 角色总览

| 角色 | 描述 | 使用端 | 权限范围 |
|------|------|--------|----------|
| 厨师 | 系统管理员，负责菜品维护、订单处理、数据查看 | Web后台 | 全部功能 |
| 客人 | 家庭成员，通过小程序点菜、支付、查看消费 | 微信小程序 | 个人相关功能 |

### 2.2 厨师（管理员）

**基本信息**：
- 年龄：35-55岁
- 特征：负责家庭日常饮食，关心食材成本和家人喜好
- 痛点：不知道做什么菜家人更喜欢、无法掌握家庭餐饮支出、手动管理菜品繁琐

**核心诉求**：
- 快速管理菜品（上架、下架、今日供应）
- 及时处理订单，掌握订单状态
- 通过数据看板了解经营概况

### 2.3 客人（家庭成员）

**基本信息**：
- 年龄：10-60岁
- 特征：使用小程序点菜，关心自己的消费记录
- 痛点：不知道自己花了多少、想知道自己最爱吃什么

**核心诉求**：
- 便捷浏览当日菜单
- 快速下单并支付
- 查看个人消费统计

## 3. 功能需求

### 3.1 客人端（微信小程序）

#### 3.1.1 用户授权与登录

**功能描述**：首次打开小程序时，获取微信用户昵称、头像，并绑定 openId 作为唯一标识。无需额外注册，自动创建客人账户。

**用户故事**：
> 作为客人，我希望打开小程序就能使用，不需要复杂的注册流程。

**功能详单**：

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 微信授权 | 获取昵称、头像、openId | P0 |
| 自动注册 | openId 不存在时自动创建账户 | P0 |
| 会话保持 | 退出后再次打开无需重新授权 | P1 |

#### 3.1.2 菜单浏览

**功能描述**：展示当日可点菜品列表，支持按分类筛选。每个菜品显示：图片、名称、价格、简介、是否售罄。

**用户故事**：
> 作为客人，我希望看到今天的菜单，了解有哪些菜可以点。

**功能详单**：

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 分类筛选 | 按凉菜、热菜、主食、饮料等分类筛选 | P0 |
| 菜品列表 | 展示今日供应的菜品 | P0 |
| 菜品详情 | 点击查看菜品详细信息 | P1 |
| 售罄标识 | 售罄菜品显示不可点击状态 | P0 |

#### 3.1.3 点菜与购物车

**功能描述**：点击菜品可查看详情，并选择份数加入购物车。购物车内可修改份数、删除菜品，实时计算总价。

**用户故事**：
> 作为客人，我希望方便地选择菜品、调整数量，并能看到总价。

**功能详单**：

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 加入购物车 | 选择份数后加入 | P0 |
| 购物车列表 | 显示已选菜品和份数 | P0 |
| 修改份数 | 增加或减少菜品数量 | P0 |
| 删除菜品 | 从购物车移除菜品 | P0 |
| 价格汇总 | 实时计算总价 | P0 |

#### 3.1.4 下单

**功能描述**：从购物车进入确认订单页，可填写备注，提交订单后生成待支付订单。

**用户故事**：
> 作为客人，我希望下单时能添加备注（如少辣），并确认订单信息。

**功能详单**：

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 订单确认页 | 显示菜品列表、总价、备注输入框 | P0 |
| 备注功能 | 支持输入特殊要求（如少辣、不要香菜） | P1 |
| 提交订单 | 生成待支付订单 | P0 |
| 订单号 | 生成唯一订单号 | P0 |

#### 3.1.5 支付

**功能描述**：采用虚拟支付模式，点击"确认支付"即视为支付成功。后续可接入真实微信支付。

**用户故事**：
> 作为客人，我希望支付流程简单快捷，点击一下就能完成支付。

**功能详单**：

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 虚拟支付 | 点击确认支付，状态变更为已支付 | P0 |
| 支付成功通知 | 向厨师后台发送新订单通知（WebSocket） | P1 |
| 微信支付（预留） | 后续接入真实微信支付 | P2 |

#### 3.1.6 订单管理

**功能描述**：我的订单列表展示所有历史订单，按时间倒序。订单详情显示当前状态，支持取消待支付订单。

**用户故事**：
> 作为客人，我希望查看我的订单历史和当前状态，并能取消未支付的订单。

**功能详单**：

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 订单列表 | 展示所有历史订单，按时间倒序 | P0 |
| 订单状态展示 | 显示待支付、已支付、制作中、已完成、已取消 | P0 |
| 订单详情 | 查看订单完整信息（菜品、金额、备注） | P0 |
| 取消订单 | 取消待支付订单 | P0 |

#### 3.1.7 个人中心

**功能描述**：展示微信昵称、头像，查看累计消费金额、订单数量等简单统计。

**用户故事**：
> 作为客人，我希望在个人中心看到我的消费统计和最爱菜品。

**功能详单**：

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 用户信息 | 展示昵称、头像 | P0 |
| 累计消费 | 所有已完成订单总金额 | P0 |
| 累计订单 | 历史订单总数 | P1 |
| 本月消费 | 当月消费金额和订单数 | P1 |
| 最爱菜品 | 点单次数最多的菜品 | P1 |

## 4. 业务流程

### 4.1 客人点菜流程

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│ 浏览菜单 │ → │ 加入购物 │ → │ 确认订单 │ → │  支  付  │ → │ 等待完成 │
│         │    │   车    │    │         │    │         │    │         │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
                                                                  │
                                                    ┌─────────────┘
                                                    ▼
                                             ┌─────────────┐
                                             │  订单完成   │
                                             └─────────────┘
```

**详细步骤**：

1. 客人进入小程序，浏览当日菜单
2. 选择菜品加入购物车，可调整份数
3. 进入购物车，确认菜品和数量，填写备注
4. 点击去结算，提交订单 → 生成待支付订单
5. 在订单页点击"支付"（虚拟支付），状态变为已支付
6. 厨师在后台看到新订单，开始制作 → 状态变为制作中
7. 厨师制作完成，点击完成 → 状态变为已完成
8. 客人可在小程序中查看订单状态更新

| 字段名 | 类型 | 说明 | 约束 |

### 9.3 名词解释

| 术语 | 解释 |
|------|------|
| openId | 微信用户在小程序中的唯一标识 |
