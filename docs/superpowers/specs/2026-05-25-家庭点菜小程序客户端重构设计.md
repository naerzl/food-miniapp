# 家庭点菜小程序客户端重构设计

**日期:** 2026-05-25
**状态:** 已批准
**版本:** v2.0

---

## 一、重构目标

将现有小程序聚焦为纯客户端（客人点餐），移除厨师管理端（管理端在 food-web 中），并使用 Tailwind CSS 重设计全部 6 个页面的 UI。

核心变更：
- 移除 chef 分包及相关代码
- 引入 Tailwind CSS 替代 SCSS
- 用 frontend-design 重新设计 6 个客人端页面

## 三、Store 设计

### 3.1 authStore（简化版）

```typescript
interface AuthState {
  isLogin: boolean
  token: string | null
  userInfo: UserInfo | null
}

interface AuthActions {
  login: (token: string, userInfo: UserInfo) => void
  logout: () => void
  updateUserInfo: (userInfo: UserInfo) => void
}
```

**变更:** 移除 `role` 字段，客人端无需角色区分。

### 3.2 cartStore（保持不变）

```typescript
interface CartItem {
  dishId: number | string
  dishName: string
  price: number
  quantity: number
  image?: string
}

interface CartState { items: CartItem[] }

interface CartActions {
  addToCart: (item: CartItem) => void
  removeFromCart: (dishId: number | string) => void
  updateQuantity: (dishId: number | string, quantity: number) => void
  clearCart: () => void
  setCart: (items: CartItem[]) => void
}
```

## 四、页面设计

### 4.1 菜单页 (menu) — TabBar 首页

- 顶部横向滚动分类标签
- 菜品卡片：图片 + 菜名 + 描述 + 价格 + 加购按钮
- 右下角购物车悬浮按钮（数量角标）
- 状态：加载骨架屏 / 空菜品占位 / 售罄灰显

### 4.2 购物车页 (cart) — TabBar

- 商品列表：图片、名称、单价、数量调节器、小计
- 底部结算栏：合计金额 + "去下单"
- 空状态：插画 + "去逛逛"引导
- 数量为 0 自动移除

### 4.3 确认下单页 (confirm)

- 订单明细（只读）+ 备注输入
- 金额汇总 + "确认下单"按钮
- 可返回购物车修改

### 4.4 订单列表页 (orders) — TabBar

- 订单卡片：订单号、状态标签、菜品摘要、金额、时间
- 下拉刷新 + 空状态占位

### 4.5 订单详情页 (order-detail)

- 状态进度展示 + 菜品明细 + 金额 + 备注 + 时间
- 取消订单（仅待支付状态）

### 4.6 个人中心页 (profile) — TabBar

- 头像 + 昵称 + 我的订单入口 + 关于信息

## 五、API 层

保留 6 个领域模块（移除 statistics）：

- `authApi` — 微信登录
- `userApi` — 用户信息
- `categoryApi` — 菜品分类
- `dishApi` — 菜品查询
- `orderApi` — 订单 CRUD

统一通过 `request.ts` 管理：Base URL 配置、Authorization 注入、401 跳转、统一错误处理。

## 六、样式系统（Tailwind CSS）

### 6.1 配置

```javascript
// tailwind.config.js
module.exports = {
  content: ['./src/**/*.{js,jsx,ts,tsx}'],
  theme: {
    extend: {
      colors: {
        // 由 frontend-design 确定具体色板
      },
    },
  },
  corePlugins: {
    preflight: false, // 避免与 Taro 组件默认样式冲突
  },
}
```

### 6.2 迁移策略

1. 安装并配置 Tailwind CSS + PostCSS
2. 创建 `src/styles/tailwind.scss` 作为 Tailwind 入口
3. 逐页将 SCSS 样式替换为 Tailwind 原子类
4. 提取通用样式为组件（EmptyState、Skeleton、DishCard）
5. 删除旧 SCSS 文件和 taro-ui 依赖

## 七、数据流

```
用户操作 → Page 组件 → Zustand hook → Store（内存 + Storage 持久化）
                ↓
          API Service → food-service 后端
```

- 购物车：纯前端 Zustand persist，离线可用
- 菜品/订单：`useDidShow` 时拉取最新数据
- 认证：微信登录 → token → Storage → 请求拦截器自动携带

## 八、错误处理

| 场景 | 处理 |
|------|------|
| 网络异常 | Toast "网络异常，请稍后重试" |
| 401 未授权 | 清除 token，跳转首页重新登录 |
| 菜品售罄 | Toast 提示，按钮置灰 |
| 下单失败 | Toast 提示原因，留在确认页可重试 |
| 空数据 | 各页面展示空状态占位组件 |
